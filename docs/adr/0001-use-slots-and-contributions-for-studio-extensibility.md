# Use Slots and Contributions for Studio extensibility

Studio extensibility uses Slots and Contributions as the common model for host-owned and module-owned extension targets. A Slot Owner defines the accepted contribution shape and local admission rules, while host policy always has final veto power over availability. This keeps routes, widgets, panels, diagnostics, Weaver tools, result renderers, and future nested module extension points under one governance model instead of separate incompatible registries.
