# Keep Dashboard and Diagnostics as separate Slots

Dashboard and Diagnostics remain separate Slots because they answer different user questions. Dashboard hosts workspace summary Contributions that help users understand what to know or do next, while Diagnostics hosts troubleshooting and observability Contributions that show what is unhealthy, failing, noisy, slow, or operationally significant. Module Load Status belongs to module management rather than Diagnostics so the Diagnostics Slot does not become a dumping ground for administration state.
