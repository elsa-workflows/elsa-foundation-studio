Foundation runtime

A runtime built for durable behavior.

  • Execution spine: scheduler queues, drain, single-writer execution, activity invocation, completion, and continuation scheduling.
  • Durability: checkpoint commit boundaries, post-commit outbox, state projections, recovery scanning, and domain retry policy.
  • Control flow: bookmarks, resume, volatile waits, timers, triggers, HTTP endpoints, flowcharts, sequences, and composites.
  • Operational state: incidents, holds, pause boundaries, execution liveness, scheduling provenance, and inspection read models.

Platform surface

Build the next capability inside the product.

Elsa 4 is making extension work visible, governed, and increasingly self-service.

01

Opt-in features and packaged modules can contribute backend services, endpoints, ESM assets, routes, panels, and settings.

The headline

Elsa 4 has moved from a workflow engine refactor toward an inspectable, modular application platform.

The important progress is not one feature—it is the connection between authoring, executable identity, durable runtime behavior, safe evidence, Studio extensibility, and governed assistance.
